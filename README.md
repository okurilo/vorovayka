# Виджетрон

MV3-расширение Chrome для захвата DOM, связанных API-запросов и компактного экспорта виджетов без отправки данных наружу.

## Что умеет

- Выбирать DOM-элемент на странице и собирать его snapshot.
- Захватывать релевантные `fetch`/`XHR` для armed-домена.
- Показывать в рабочей области DOM preview, выбранные API и итоговый экспорт.
- Поддерживать два режима захвата: `Обычный` по умолчанию и `PRO` для ручной проверки API во Viewer.
- Записывать процесс работы с интерфейсом: пользователь задаёт название, проходит сценарий, а Виджетрон накапливает последовательность API-вызовов.
- Подсвечивать связи `DOM ↔ API`, если для capture доступен `cloneSpec`.
- Давать компактный экспорт в режимах `Всё вместе`, `API`, `OpenAPI schema`, `DOM clean`, `DOM raw`.
- Показывать отдельную страницу процесса с хронологией API и JSON-экспортом `widgetron.process-export.v1` для восстановления сценария через LLM.
- Формировать `OpenAPI schema` как OAS 3.1-совместимый `Schema Object`: с JSON Schema-ключами, `examples` вместо устаревшего `example`, корректным представлением `null`/текстовых ответов и компактными примерами только на leaf-уровне, без раздувания контейнеров `object`/`array`.
- Для leaf-полей схема может сохранять до `2-3` коротких examples; base64 и blob-like строки автоматически заменяются на маркер `base64`.
- Для `OpenAPI schema` сохраняется практический лимит на размер, но он рассчитан на реальные ответы: вложенность схемы строится глубже прежнего, а число полей объекта больше не режется агрессивно на первых уровнях.

## Основной сценарий

1. Откройте `chrome://extensions` и загрузите проект как `Load unpacked`.
2. Откройте обычную страницу по `http` или `https`.
3. В popup `Виджетрон` включите захват для текущего домена.
4. Оставьте `Обычный` режим для быстрого сценария или включите `PRO`, если хотите вручную выбрать API и открыть Viewer.
5. После перезагрузки вкладки нажмите `Выбрать элемент`.
6. В обычном режиме Виджетрон сам сохранит `DOM clean + OpenAPI schema`; в неоднозначных случаях попросит вручную отметить API.
7. В `PRO` режиме подтвердите нужные API в in-page модалке и при необходимости откройте рабочую область.

## UX flow

- `Обычный` режим ведёт по короткому пути: разрешите сбор на сайте, выберите нужный блок, дождитесь сохранения контекста.
- Если система не уверена, какие API относятся к блоку, она попросит просто отметить нужные источники без перехода в Viewer.
- `PRO` режим сохраняет тот же runtime-сценарий, но оставляет больше ручного контроля над выбором API и открытием рабочей области.

## Запись процесса

1. Откройте обычную страницу по `http` или `https` и включите сбор запросов для текущего домена.
2. В popup откройте режим `Процесс`, задайте название и нажмите `Старт процесса`.
3. Выполните нужный сценарий в интерфейсе сайта.
4. Откройте popup и нажмите `Стоп процесса`; активная запись автоматически покажет режим `Процесс`.
5. Нажмите `Страница процесса`, чтобы увидеть хронологию API-вызовов и скопировать LLM-компактный экспорт.

Запись процесса использует тот же локальный и отредактированный сетевой поток, что и обычный capture: чувствительные заголовки удаляются, данные не отправляются наружу, а запись работает только для armed-домена.
Копируемый экспорт процесса не включает сырые `responseBody`, headers и initiator stack: они остаются в локальном просмотре деталей, а JSON для нейронки содержит порядок API, короткие request body, response preview и схемы ответов.

## Ограничения

- Захват сети работает только для явно armed-домена.
- Данные не отправляются наружу: используются только локальные механизмы расширения.
- `latestCapture` остаётся временным сообщением и очищается после чтения или по TTL.
- Запись процесса хранится локально в IndexedDB расширения и ограничена последними API-событиями, чтобы MVP не превращался в полноценный сетевой инспектор.
- Если полный capture недоступен и используется compact fallback, это должно быть явно видно в UI.

## Контекст для основного Виджетрона

Обычный режим сохраняет в browser context компактный payload, на который должен опираться основной Виджетрон:

- `dom.cleanHtml` — главный источник структуры выбранного блока.
- `apiSchema` — структурная типизация связанных API, а не гарантия конкретных значений.
- `apiResolution.strategy`, `autoSelectedIds`, `ambiguousCandidates` — метаданные того, как расширение выбрало API.
- Если в payload несколько API, основной Виджетрон должен объяснять, какой endpoint за что отвечает, а не схлопывать их в один источник без оснований.
- Если сырых request/response body нет, основной Виджетрон не должен требовать их от расширения.
